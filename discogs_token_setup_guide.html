🎯 为什么需要配置Token？

  • 数据丰富度：获取完整的专辑信息、曲目列表、发行详情
  • 数据准确性：来自Discogs官方数据库的权威信息
  • 覆盖范围：访问数百万张专辑的详细信息
  • 实时更新：获取最新的音乐发行信息

📝 获取Token的详细步骤

  1. 访问Discogs开发者页面
    🌐打开Discogs开发者设置

    如果没有Discogs账号，需要先注册一个免费账号

  2. 创建新应用
    点击 "Create an App" 按钮，填写以下信息：
    • App Name: BananaRecordApp（或你喜欢的名称）
    • Description: 音乐唱片浏览应用
    • Website URL: http://localhost:8080（开发用）
  3. 生成Personal Access Token
    在应用详情页面，找到 "Generate new token" 按钮并点击
    📸 这里会显示你的Personal Access Token
    格式类似：AbCdEfGhIjKlMnOpQrStUvWxYz1234567890
  4. 复制Token
    ⚠️ 重要：Token只会显示一次，请立即复制并保存！

⚙️ 配置Token到应用

1
打开配置文件
找到文件：entry/src/main/ets/common/ApiConfig.ets
2
修改Token配置
找到第9行，将你的Token替换示例Token：
// 修改前 private static DISCOGS_TOKEN = 'NavEcjohrJttDkWwMzWNKEIRmyLsecfRSSWoCIVA'; // 修改后（替换为你的Token） private static DISCOGS_TOKEN = '你的_Discogs_Token_在这里';
3
保存并重新编译
保存文件后，重新编译应用即可生效

🧪 验证配置是否成功

✅配置成功的标志

  • 点击唱片时能看到详细的专辑信息
  • 控制台显示 "✅ 成功从Discogs API获取唱片详情"
  • 唱片详情页显示完整的曲目列表和发行信息

⚠️常见问题

  • Token无效：检查Token是否完整复制，没有多余空格
  • 网络错误：确保网络连接正常，可以访问Discogs API
  • 配额限制：Discogs API有请求频率限制，过于频繁请求可能被限制

🔒 安全提醒

  • 不要分享Token：Token是你的个人凭证，不要在公开场合分享
  • 定期更新：如果Token泄露，及时在Discogs后台重新生成
  • 本地开发：当前配置仅用于本地开发，生产环境需要额外的安全措施
